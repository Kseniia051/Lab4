## Ответы на вопросы

### 1. Какой подход показался удобнее для этой задачи и почему?
**Bootstrap (компонентный подход)** оказался удобнее для этой задачи, потому что:  

- Быстрое прототипирование с готовыми компонентами (`navbar`, `card`, `btn`).  
- Минимум ручной работы с классами и стилями.  
- Элементы сразу выглядят аккуратно и адаптивно.  

**Tailwind (utility-first)** потребовал больше времени на подбор утилитарных классов, но дал полную свободу дизайна и точное соответствие желаемому виду.

---

### 2. Пример кода одного и того же элемента (кнопки)

**Bootstrap:**
```html
<a href="#" class="btn btn-primary">Подробнее</a>
```

Tailwind CSS:
```
<a href="#" class="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600 text-center">Подробнее</a>
```

Ключевое различие:

Bootstrap использует готовый компонент с внутренними стилями.

Tailwind требует задания всех стилей через утилитарные классы прямо в HTML (цвет, отступы, закругления, hover).

---

### 3. Сложности при работе с каждой библиотекой

Bootstrap:
Труднее кастомизировать уникальные элементы.
Без переопределения стилей дизайн выглядит стандартно.

Tailwind CSS:
HTML содержит много классов, что "замусоривает" код.
Порог входа выше — нужно изучить названия всех утилитарных классов.
Дольше собирать даже простые компоненты.
